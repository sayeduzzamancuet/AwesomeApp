## 💡 Pro Tips

1. **Content Updates**
   - Edit `portfolio.data.ts` - TypeScript will validate
   - Changes auto-reload in dev server

2. **Styling**
   - Use CSS variables from `styles.css`
   - Component styles are scoped (won't leak)
   - Mobile-first approach

3. **Adding Features**
   - Follow component pattern in ARCHITECTURE.md
   - Each component: .ts, .html, .css, .spec.ts
   - Import in portfolio-layout component

4. **Testing**
   - Run: `npm test`
   - Examples provided in .spec.ts files
   - Aim for >80% coverage

5. **Performance**
   - Keep bundle small
   - Lazy load images (done by default)
   - Monitor with Lighthouse
